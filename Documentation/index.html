Introduction

Thank you for purchasing Pouch — a complete, non-custodial cryptocurrency wallet ecosystem. This package includes three fully functional applications:

Mobile App

React Native (Expo SDK 54) wallet with multi-chain support, token swaps, WalletConnect, biometric security, and push notifications.

Backend Server

Express.js API server with Redis caching, MongoDB persistence, CoinGecko proxy, and Alchemy webhook-based transaction notifications.

Landing Website

Next.js 16 marketing site with responsive design, early-access waitlist signup, and Firebase integration.

Monorepo

Turborepo-managed workspace with shared Docker infrastructure, unified dev scripts, and parallel builds.

What's Included

  • Full source code for all three applications
  • Turborepo monorepo configuration with npm workspaces
  • Docker Compose setup for Redis and MongoDB

Installation

Estimated time: 10-15 minutes for full setup including Docker containers.
1

Extract the Package

Unzip the downloaded file and open the pouch-monorepo folder in your terminal:

cd pouch-monorepo
2

Install Dependencies

Install all workspace dependencies from the root. npm workspaces will handle all three apps:

npm install

This installs dependencies for the mobile app, server, and website in one command.

3

Create Environment Files

Run the setup script to copy .env.example templates into actual .env files:

npm run setup

This creates:

  • apps/server/.env — Server configuration
  • apps/web/.env.local — Website configuration
  • apps/mobile/.env — Mobile configuration
4

Configure Environment Variables

Open each .env file and fill in your values. See the Environment Variables section for a complete reference. At minimum you need:

  • Server: MONGODB_URI and REDIS_URL (defaults work with Docker)
  • Website: FIREBASE_PROJECT_ID, FIREBASE_CLIENT_EMAIL, FIREBASE_PRIVATE_KEY
  • Mobile: Alchemy API key in the app's config files
5

Start Docker Containers

Start Redis and MongoDB using Docker Compose:

npm run docker:up

Verify they're running:

docker ps
# Should show pouch-redis (port 6379) and pouch-mongodb (port 27017)
6

Start All Dev Servers

Launch all three apps simultaneously with Turborepo's interactive TUI:

npm run dev

This starts:

  • Mobile: Expo dev server at http://localhost:8081
  • Server: Express API at http://localhost:3001
  • Web: Next.js at http://localhost:3000
7

Run the Mobile App

Open the Expo Go app on your phone and scan the QR code shown in the terminal. For first run, clear the Metro cache:

cd apps/mobile && npx expo start --clear
Done! You should now have all three applications running. Visit http://localhost:3001/health to verify the server is connected to Redis and MongoDB.

Running Individual Apps

# Run only the mobile app
npm run dev:mobile

# Run only the server
npm run dev:server

# Run only the website
npm run dev:web

# Run any Turbo task for a specific workspace
npx turbo run build --filter=@pouch/web
npx turbo run lint --filter=@pouch/mobile

Configuration

Server URL (Mobile → Backend Connection)

The mobile app needs to know where the backend server is running. This is configured in two places:

During Development

In development mode (__DEV__ === true), the app automatically connects to http://localhost:3001. No changes needed if you're running everything locally with npm run dev.

If you're testing on a physical device (not a simulator), localhost won't work. Set the env var to your computer's LAN IP:

# apps/mobile/.env
EXPO_PUBLIC_CACHE_SERVER_URL=http://192.168.1.100:3001
Find your LAN IP: Run ifconfig | grep "inet " on macOS or hostname -I on Linux. Use the 192.168.x.x or 10.x.x.x address.

For Production

Open apps/mobile/constants/api.ts and change PROD_URL to your deployed server's URL:

// constants/api.ts
const DEV_URL = 'http://localhost:3001';
const PROD_URL = 'https://api.yourdomain.com';  // ← Change this

export const CACHE_SERVER_URL =
  process.env.EXPO_PUBLIC_CACHE_SERVER_URL || (__DEV__ ? DEV_URL : PROD_URL);

Alternatively, you can set the env var in your EAS build profile without touching the code:

// eas.json
{
  "build": {
    "production": {
      "env": {
        "EXPO_PUBLIC_CACHE_SERVER_URL": "https://api.yourdomain.com"
      }
    }
  }
}
Important: The production URL must use HTTPS. Make sure your server has SSL configured (see Deploy Server > Nginx + Certbot).

Alchemy Setup

Alchemy provides blockchain data (token balances, transactions, gas estimation) for the mobile app.

  1. Create a free account at alchemy.com
  2. Create an app for each network you want to support (Ethereum, Polygon, Arbitrum, Optimism, Base)
  3. Copy the API key from your Alchemy dashboard
  4. Open apps/mobile/services/blockchain/alchemy-client.ts and set your API key

Firebase Setup (Website Only)

Firebase is used for the email waitlist feature on the landing website.

  1. Go to the Firebase Console and create a project
  2. Enable Firestore Database
  3. Go to Project Settings > Service Accounts > Generate New Private Key
  4. Copy the values into apps/web/.env.local: 
    FIREBASE_PROJECT_ID=your-project-id
FIREBASE_CLIENT_EMAIL=firebase-adminsdk-xxxxx@your-project.iam.gserviceaccount.com
FIREBASE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"

Push Notifications (Optional)

Transaction push notifications require Alchemy webhooks. This is optional — the app works fully without it.

  1. In the Alchemy dashboard, go to Webhooks and create an ADDRESS_ACTIVITY webhook
  2. Set the webhook URL to your deployed server's /webhooks/alchemy endpoint
  3. Copy the webhook ID and signing key into apps/server/.env

Firebase Setup (Mobile — Push Notifications)

The mobile app uses Firebase Cloud Messaging (FCM) via Expo's notification system. This requires platform-specific Firebase config files. The website uses Firebase Admin SDK separately (see Configuration).

Note: Push notifications only work in development builds or production — not in Expo Go. You will need to create a development build with eas build to test them.

Create a Firebase Project

  1. Go to the Firebase Console
  2. Click Add project and follow the wizard (you can disable Google Analytics)
  3. Once created, you'll add platform-specific apps in the next steps

Android Setup

Step 1: Register Android App in Firebase

  1. In the Firebase Console, click Add app > Android
  2. Enter the Android package name: com.mithium.pouch
    Must match exactly: This must match the "package" field in apps/mobile/app.json under expo.android.package. If you change the package name, update both places.
  3. Click Register app
  4. Download the google-services.json file

Step 2: Place the Config File

Copy the downloaded google-services.json into the mobile app root:

cp ~/Downloads/google-services.json apps/mobile/google-services.json

The file is already referenced in app.json:

{
  "expo": {
    "android": {
      "googleServicesFile": "./google-services.json",
      "package": "com.mithium.pouch"
    }
  }
}
Security: The google-services.json file is gitignored by default. Never commit it to a public repository. It contains your Firebase project configuration including API keys.

Step 3: Build and Test

# Create a development build for Android
cd apps/mobile
eas build --platform android --profile development

# Install on device/emulator and test push notifications

Customizing the Package Name

If you want to use your own Android package name (e.g., com.yourcompany.pouch), update these locations:

  1. apps/mobile/app.json — Change expo.android.package
  2. Firebase Console — Register a new Android app with your package name and download a fresh google-services.json

iOS Setup

Step 1: Register iOS App in Firebase

  1. In the Firebase Console, click Add app > iOS
  2. Enter the iOS bundle ID: com.nawabkibria12.pouch
    Must match exactly: This must match the "bundleIdentifier" field in apps/mobile/app.json under expo.ios.bundleIdentifier. If you change the bundle ID, update both places.
  3. Click Register app
  4. Download the GoogleService-Info.plist file

Step 2: Place the Config File

Copy the downloaded plist into the mobile app root:

cp ~/Downloads/GoogleService-Info.plist apps/mobile/GoogleService-Info.plist

Then reference it in app.json by adding the googleServicesFile field under ios:

{
  "expo": {
    "ios": {
      "googleServicesFile": "./GoogleService-Info.plist",
      "bundleIdentifier": "com.nawabkibria12.pouch",
      "supportsTablet": true,
      "infoPlist": {
        "NSFaceIDUsageDescription": "Allow Pouch to use Face ID to unlock your wallet"
      }
    }
  }
}

Step 3: Apple Push Notification Key (APNs)

iOS push notifications require an Apple Push Notification service (APNs) key. This is a .p8 file that you upload to Firebase.

1

Generate the APNs Key

  1. Go to Apple Developer > Keys
  2. Click the + button to create a new key
  3. Enter a name (e.g., "Pouch Push Notifications")
  4. Check Apple Push Notifications service (APNs)
  5. Click Continue, then Register
  6. Download the .p8 key file — save it securely, you can only download it once
  7. Note the Key ID shown on the page
2

Upload to Firebase

  1. In the Firebase Console, go to Project Settings > Cloud Messaging
  2. Under Apple app configuration, click Upload next to APNs Authentication Key
  3. Upload the .p8 file you just downloaded
  4. Enter the Key ID (from the Apple Developer portal)
  5. Enter your Team ID (found at Apple Developer > Membership)
3

Build and Test

# Create a development build for iOS
cd apps/mobile
eas build --platform ios --profile development

# Install on simulator or physical device
# Push notifications only work on physical iOS devices
Important: iOS push notifications do not work on the iOS Simulator. You must test on a physical device. Also, you need an active Apple Developer Program membership ($99/year).

Customizing the Bundle Identifier

If you want to use your own iOS bundle ID (e.g., com.yourcompany.pouch), update these locations:

  1. apps/mobile/app.json — Change expo.ios.bundleIdentifier
  2. Firebase Console — Register a new iOS app with your bundle ID and download a fresh GoogleService-Info.plist
  3. Apple Developer Portal — Create a new App ID with the matching bundle identifier

EAS Project Configuration

The app.json file references an EAS project ID. You must update this with your own Expo account's project:

# Link your Expo account to this project
cd apps/mobile
eas init

# This updates the projectId in app.json automatically

The extra.eas.projectId in app.json will be updated to your Expo account's project. This is required for push notification token generation and EAS builds.

Wallet Management

  • Create Wallet: Generate a new BIP39 mnemonic (12/24 words) with HD wallet derivation (BIP44)
  • Import Wallet: Restore existing wallet from recovery phrase
  • Multiple Accounts: Derive additional accounts from the same mnemonic
  • Account Switching: Quick switch between accounts from any screen
  • Secure Storage: Private keys encrypted with AES-256 and stored in Expo Secure Store

Portfolio & Prices

  • Real-Time Prices: Live cryptocurrency prices via CoinGecko (cached by the backend server)
  • Price Charts: Interactive charts with 1D, 7D, 30D, 1Y, and all-time timeframes
  • Portfolio Value: Total USD portfolio value with 24h change
  • Allocation Chart: Pie chart visualization of asset distribution
  • Diversification Score: Portfolio health metric (0-100)
  • Best/Worst Performers: Highlights top and bottom performing assets

Send & Receive

  • Send Native Tokens: ETH, MATIC, and other native currencies
  • Send ERC-20 Tokens: Any supported ERC-20 token
  • Address Book: Save frequently-used recipients as contacts
  • Gas Estimation: Real-time gas fees with Standard / Fast / Instant options
  • QR Code Receive: Generate QR codes for your wallet address
  • Transaction History: Per-token and overall transaction logs

Token Swaps

  • DEX Integration: Uniswap V3 for on-chain token swaps
  • Live Quotes: Real-time swap quotes with 15-second refresh
  • Slippage Control: Presets (0.5%, 1%, 3%) and custom slippage tolerance
  • Price Impact: Visual indicator of trade impact on pool
  • Multi-Network: Swap on any supported EVM chain with Uniswap deployment
  • Swap History: Track all past swaps

WalletConnect

  • WalletConnect v2: Connect to any dApp via QR code scanning
  • Session Management: View, manage, and disconnect active sessions
  • Request Signing: Support for personal_sign, eth_sign, eth_signTypedData_v4, eth_sendTransaction
  • Multiple Sessions: Connect to multiple dApps simultaneously

Notifications

  • Push Notifications: Real-time alerts for incoming/outgoing transactions
  • Alchemy Webhooks: Blockchain monitoring for registered wallet addresses
  • Notification History: Paginated history with read/unread status
  • Multi-Address: Monitor all accounts in your wallet

Security

  • Non-Custodial: Private keys never leave the device
  • PIN Protection: 6-digit PIN with exponential lockout after failed attempts
  • Biometric Auth: Face ID / Touch ID support
  • Auto-Lock: Configurable timeout (immediate, 1min, 5min, 15min)
  • Encrypted Storage: AES-256 encryption for all sensitive data
  • Recovery Phrase: Biometric-protected recovery phrase viewer

Linux VPS with PM2

Deploy to any Ubuntu/Debian VPS (DigitalOcean, Hetzner, Linode, AWS EC2, etc.) using PM2 for process management.

1

Install Prerequisites on the Server

# Update packages
sudo apt update && sudo apt upgrade -y

# Install Node.js 20 (via NodeSource)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Install PM2 globally
sudo npm install -g pm2

# Install Redis
sudo apt install -y redis-server
sudo systemctl enable redis-server

# Install MongoDB (Ubuntu 22.04 example)
# See https://www.mongodb.com/docs/manual/tutorial/install-mongodb-on-ubuntu/
curl -fsSL https://www.mongodb.org/static/pgp/server-7.0.asc | \
  sudo gpg -o /usr/share/keyrings/mongodb-server-7.0.gpg --dearmor
echo "deb [ signed-by=/usr/share/keyrings/mongodb-server-7.0.gpg ] https://repo.mongodb.org/apt/ubuntu jammy/mongodb-org/7.0 multiverse" | \
  sudo tee /etc/apt/sources.list.d/mongodb-org-7.0.list
sudo apt update
sudo apt install -y mongodb-org
sudo systemctl enable mongod
sudo systemctl start mongod
2

Upload and Install the Server App

# On your VPS, create a directory and upload the server files
mkdir -p ~/pouch-server
cd ~/pouch-server

# Option A: Clone from your repo and extract the server
git clone https://github.com/your-username/pouch-monorepo.git .
cd apps/server

# Option B: Upload just the apps/server directory via scp
# (from your local machine)
# scp -r apps/server/ user@your-server-ip:~/pouch-server/

# Install dependencies
npm install --production
3

Configure Environment Variables

# Copy and edit the .env file
cp .env.example .env
nano .env

Set production values:

# .env
PORT=3001
REDIS_URL=redis://localhost:6379
MONGODB_URI=mongodb://localhost:27017/pouch

# Add your CoinGecko, Alchemy, and Expo keys
COINGECKO_API_KEY=your_key_here
ALCHEMY_AUTH_TOKEN=your_token_here
ALCHEMY_WEBHOOK_1_ID=your_webhook_id
ALCHEMY_WEBHOOK_1_SIGNING_KEY=your_signing_key
ALCHEMY_WEBHOOK_1_NETWORK=ETH_MAINNET
4

Start with PM2

# Start the server with PM2
pm2 start src/index.js --name pouch-server

# Verify it's running
pm2 status
pm2 logs pouch-server

# Test the health endpoint
curl http://localhost:3001/health
5

Auto-Start on Reboot

# Generate the PM2 startup script
pm2 startup

# Copy and run the command it outputs (will look like):
# sudo env PATH=$PATH:/usr/bin pm2 startup systemd -u your-user --hp /home/your-user

# Save the current process list
pm2 save

PM2 will now automatically restart the server if it crashes or if the VPS reboots.

6

Set Up Nginx Reverse Proxy (Recommended)

# Install Nginx
sudo apt install -y nginx

# Create a site config
sudo nano /etc/nginx/sites-available/pouch-server

Paste this configuration:

server {
    listen 80;
    server_name api.yourdomain.com;

    location / {
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
# Enable the site and restart Nginx
sudo ln -s /etc/nginx/sites-available/pouch-server /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
7

Add SSL with Certbot (Free HTTPS)

# Install Certbot
sudo apt install -y certbot python3-certbot-nginx

# Generate and install SSL certificate
sudo certbot --nginx -d api.yourdomain.com

# Auto-renewal is set up automatically — verify with:
sudo certbot renew --dry-run
